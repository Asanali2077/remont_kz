## Решения принятые в дизайне

| Вопрос | Решение |
|--------|---------|
| Библиотека | `next-intl` |
| Роутинг | URL-based (`/ru/...`, `/en/...`, `/kk/...`) |

### URL-роутинг

```
/           → redirect → /ru
/ru         → главная на русском
/en/repair  → каталог на английском
/kk/company → dashboard компании на казахском
```

### Middleware

Объединить существующий CORS-middleware с `next-intl` middleware:

```typescript
// middleware.ts
import createMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';

const intlMiddleware = createMiddleware(routing);

export function middleware(request: NextRequest) {
  // 1. CORS headers (существующая логика)
  // 2. next-intl locale detection + redirect
  return intlMiddleware(request);
}
```

Логика определения языка:
1. Проверить URL — есть ли `/ru`, `/en`, `/kk`
2. Если нет — читать cookie `NEXT_LOCALE`
3. Если нет — читать `Accept-Language` заголовок
4. Fallback → `ru`

## Переключатель языка

### Расположение
В `MainNavbar` — рядом с кнопкой темы (справа).

### Внешний вид
```
[ RU ]  EN  ҚАЗ
```
Активный язык — выделен (bold или подчёркивание). Три кнопки без dropdown.

### Логика
```typescript
// При клике на EN:
// /ru/repair?city=Almaty → /en/repair?city=Almaty
// pathname и searchParams сохраняются
```

Реализуется через `usePathname` + `useRouter` из `next-intl/navigation`.

## Порядок реализации (для следующей сессии)

### Этап 1 — Инфраструктура (без переводов)
1. Установить `next-intl`
2. Создать `i18n/config.ts`, `i18n/routing.ts`, `i18n/request.ts`
3. Обновить `middleware.ts`
4. Создать `app/[locale]/` — переместить все страницы
5. Обновить `app/[locale]/layout.tsx` — добавить `NextIntlClientProvider`
6. Проверить что роутинг работает (`/ru/repair` открывается)

### Этап 2 — Переводы ru.json
7. Создать `messages/ru.json` со всеми строками
8. Заменить хардкод-строки в компонентах на `t('key')`
9. Начать с `common` + `nav` + `auth` (самые переиспользуемые)
10. Затем по страницам: home → repair → service → requests → company → admin

### Этап 3 — Переводы en.json + kk.json
11. Скопировать структуру ru.json
12. Перевести en.json (английский)
13. Перевести kk.json (казахский кириллица)

### Этап 4 — Переключатель + финализация
14. Добавить `LanguageSwitcher` компонент в Navbar
15. Обновить все `<Link>` и `router.push()` — использовать locale-aware версии из `next-intl/navigation`
16. Проверить `lang` атрибут в `<html>` — должен меняться динамически
17. type-check + lint
